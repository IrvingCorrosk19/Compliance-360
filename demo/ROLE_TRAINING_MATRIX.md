# ROLE_TRAINING_MATRIX

## Objetivo
Definir rutas de aprendizaje por rol para que cualquier persona se familiarice con Compliance 360 de forma práctica y funcional.

## Fase 1 (implementada en demo)

## Quality Manager

- **Ruta recomendada:** Dashboard -> Document Management -> CAPA -> Risks -> Reports
- **Aprende a:**
  - revisar estado de cumplimiento,
  - aprobar/rechazar documentos,
  - priorizar acciones correctivas,
  - dar seguimiento a riesgos,
  - presentar resultados ejecutivos.

## Document Controller

- **Ruta recomendada:** Dashboard -> Document Management -> Approval Workflows -> Reports
- **Aprende a:**
  - crear documentos controlados,
  - clasificar información,
  - preparar flujo de aprobación,
  - publicar evidencia y generar reportes.

## Fase 2 (implementada en demo)

## Auditor

- **Ruta recomendada:** Dashboard -> Audits -> CAPA -> Reports
- **Aprende a:**
  - planificar auditorías,
  - documentar hallazgos,
  - conectar acciones correctivas,
  - reportar estado para dirección.

## CAPA Manager

- **Ruta recomendada:** Dashboard -> CAPA -> Risks -> Indicators -> Reports
- **Aprende a:**
  - gestionar acciones correctivas/preventivas,
  - priorizar por riesgo,
  - medir eficacia con indicadores.

## Risk Manager

- **Ruta recomendada:** Dashboard -> Risks -> Indicators -> Reports
- **Aprende a:**
  - registrar riesgos,
  - evaluar impacto/probabilidad,
  - definir tratamiento y seguimiento.

## Fase 3 (implementada en demo)

## Reporting Manager

- **Ruta recomendada:** Dashboard -> Reports -> Indicators -> Document Management
- **Aprende a:**
  - consolidar datos,
  - construir reportes ejecutivos,
  - preparar exportables para comité.

## Tenant Admin

- **Ruta recomendada:** Dashboard -> Tenant Administration -> Users -> Roles -> Permissions -> Settings
- **Aprende a:**
  - administrar acceso,
  - aplicar RBAC,
  - mantener gobernanza del tenant.

## Viewer

- **Ruta recomendada:** Dashboard -> Document Management -> CAPA -> Risks -> Reports
- **Aprende a:**
  - consultar información transversal,
  - interpretar estado operativo,
  - apoyar decisiones sin editar datos.
