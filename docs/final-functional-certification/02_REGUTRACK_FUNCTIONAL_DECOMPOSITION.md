## 2. Workbook topology

```
REGUTRACK 02JUN26 MG.xlsx
├── CTT REGISTROS          → Portfolio (all products / registrations)
├── CTT REGISTROS (2)      → Same schema; alternate view / export
├── CTT REGISTROS TUBERIA  → Pipeline (status-filtered cases)
├── DOCUMENTACION          → Manufacturer certificate vault
└── CTT LICENCIAS OP       → Operating licenses + renewal checklist
```

Three registration sheets share **identical column structure** (88 headers, row 1 = header). System mapping:

| Excel sheet | System representation | Primary UI |
|-------------|----------------------|------------|
| CTT REGISTROS | `MedicalDeviceProduct` + optional active `SanitaryRegistration` | Portafolio |
| CTT REGISTROS (2) | Same entities (duplicate export) | Portafolio / Import |
| CTT REGISTROS TUBERIA | `RegistrationDossier` filtered by workflow status | Pipeline |
| DOCUMENTACION | `ManufacturerProfile` + `ManufacturerCertificate` | Fabricantes |
| CTT LICENCIAS OP | `OperatingLicense` + `LicenseRenewalCase` | Licencias |

## 3. Domain A — CTT REGISTROS (columns 1–88)

### 3.1 Critical band (columns 1–55)

Used for certification exit criteria. Sub-domains:

| Col range | Sub-domain | Entity focus |
|-----------|------------|--------------|
| 1–9 | Product identity | Country, category, brand, name, description, catalog, manufacturer, distributor, initiative |
| 10–11 | Technical & authority | Ficha técnica ref, issuing authority |
| 12–15 | Registration metadata | CT/RS number, issue date, supplier count, risk class |
| 16–17 | Process | Process type, form/pack reference |
| 18–39 | Document checklist | 22 pack items (`REGUTRACK-PA-DEFAULT`) |
| 40–43 | Manufacturer doc timeline | Factory request, est/max/received dates |
| 44 | Alerts | Derived alert flags |
| 45–50 | Submission & approval dates | Assembled, est/submit, submit, observation, est/actual approval |
| 51–55 | Business metadata | Sales input, priority, opportunity, comments, expiry |

### 3.2 History band (columns 56–87)

Excel stores 32 “Fecha de Actualización” columns. System equivalent: **`DossierHistoryEvent`** typed event stream on status transitions. Certification tests one transition → one history row (not 32 separate fields).

### 3.3 Line metadata (column 88)

`SourceLineNumber` for import reconciliation (`Línea`).

### 3.4 Decomposition JSON pointers

## 4. Domain B — CTT REGISTROS TUBERIA

| Pipeline column (UI) | Typical Excel tubería meaning |

| UnderAuthorityReview | Authority reviewing |
| Observed | Observation issued |
| CorrectingObservation | Fixing obs |
| Approved | CT/RS granted |
| Vencido | Expired registration |
| Renovacion | Renewal in progress |
