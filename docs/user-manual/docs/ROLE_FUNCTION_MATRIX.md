# ROLE_FUNCTION_MATRIX

| Role | Purpose | Can | Cannot | Flow stage |
|------|---------|-----|--------|------------|
| Tenant Administrator | Administra el tenant: usuarios, roles y configuración general. No opera expedientes regulatorios por defecto (SoD). | Crear y desactivar usuarios (TENANT.USERS); Asignar roles (TENANT.ROLES / RBAC.MANAGE)… | Crear expedientes (sin REGULATORY.DOSSIER.CREATE); Aprobar internamente para sometimiento… | IAM · configuración · sin operación de expediente |
| Regulatory Administrator | Configura el módulo Regulatory Affairs: autoridades, packs, alertas e importación REGUTRACK. | Bootstrap regulatorio (autoridades MINSA/CSS + pack de requisitos); Importación Stage XLSX REGUTRACK (REGULATORY.CONFIGURE)… | Aprobar internamente (APPROVE_FOR_SUBMISSION); Registrar sometimiento (SUBMIT)… | Configuración previa al flujo del expediente |
| Regulatory Manager | Supervisa el pipeline, registra observaciones de autoridad y la decisión externa que crea el CT/RS. | Ver dashboard, pipeline, portafolio y aging; Actualizar expedientes (DOSSIER.UPDATE)… | Sustituir Specialist + Approver + Submitter a la vez sin override; Representar que la aprobación interna es la de MINSA/CSS… | Observación → Respuesta → Decisión externa → CT/RS → Renovación |
| Regulatory Specialist | Prepara productos y expedientes, solicita documentos, completa requisitos y responde observaciones. | Crear producto + expediente (PRODUCT.MANAGE + DOSSIER.CREATE); Transiciones de preparación: Pedir docs, Docs recibidos, Armar, Declarar técnicamente completo… | Revisar su propio expediente (PreventSelfReview / SoD); Aprobar internamente para sometimiento… | Preparación (Planning → ReadyForSubmission) y respuesta a observaciones |
| Regulatory Reviewer | Revisa técnicamente cada requisito: Aceptar o Rechazar. No prepara ni aprueba internamente. | Abrir expedientes en revisión (DOSSIER.READ + REVIEW); Aceptar requisito… | Crear productos/expedientes; Aprobar internamente para sometimiento… | Revisión técnica (antes de ReadyForSubmission / hacia Approver) |
| Regulatory Approver | Autoriza internamente que el expediente pueda ser sometido. No representa a MINSA/CSS. | Ver expedientes en estado Técnicamente completo; Presionar «Aprobar internamente para sometimiento» (APPROVE_FOR_SUBMISSION)… | Preparar el expediente; Registrar sometimiento (SeparateApproverAndSubmitter)… | Aprobación interna (ReadyForSubmission → ApprovedForSubmission) |
| Regulatory Submitter | Registra el sometimiento real ante la autoridad. No otorga la aprobación interna. | Ver expedientes «Aprobado internamente para sometimiento»; Presionar «Registrar sometimiento» (DOSSIER.SUBMIT)… | Aprobar internamente; Registrar decisión externa / CT/RS… | Sometimiento (ApprovedForSubmission → Submitted) |
| Regulatory Viewer | Solo lectura del portafolio regulatorio, pipeline y CT/RS. | Ver Dashboard, Portafolio, Pipeline, Expedientes, CT/RS, Alertas; Consultar sin modificar… | Cualquier botón de mutación (Nuevo, Aprobar, Someter, Stage, Bootstrap, etc.)… | Consulta transversal (sin etapa de escritura) |
| Quality Manager | Coordinación QMS transversal. Puede registrar la decisión externa CT/RS; no prepara expedientes. | Registrar aprobación externa / CT/RS (DOSSIER.APPROVE + REGISTRATION.MANAGE); Leer expedientes y reportes RA… | Preparar expedientes (sin CREATE/UPDATE de preparación por defecto); Sustituir al Regulatory Manager en supervisión diaria salvo su permiso de APPROVE… | Decisión externa / CT/RS (junto o en apoyo al Manager) |
