# BUG_003 — Regulatory Reviewer could create products (SoD)

| Field | Value |
|-------|-------|
| Severity | P0 |
| Status | Fixed |
| Found | 2026-07-14 CERT run |
| Area | RBAC / Regulatory Affairs |

## Symptom
`ra.rev@cert.local` (Regulatory Reviewer) `POST .../regulatory/products` returned **200** despite lacking `REGULATORY.PRODUCT.MANAGE`.

## Root cause
Friendly policy `Regulatory.Manage` used `AddAny` including `REGULATORY.REGISTRATION.MANAGE`. Reviewers hold Registration Manage for CT/RS updates after approve, so they incorrectly satisfied product/dossier write endpoints.

## Fix
1. Removed `REGULATORY.REGISTRATION.MANAGE` from `Regulatory.Manage` `AddAny`.
2. Bound `POST /products` to `REGULATORY.PRODUCT.MANAGE` explicitly.

## Verification
Re-run SOD-REV-CREATE-PRODUCT → expect **403**.
